# AXIOMS

1. Axioms are sacred.
   Accepted as truth, never challenged, amended, added to, or removed, except by a human.
2. Keep Main Green.
   Only validated work lands on main.
3. Preserve Every Attempt.
   Nothing is hard-deleted. Failed work is saved before green is restored.
4. Own the Next Move.
   The repo and roadmaps guide. The agent may follow, adapt, rewrite, or replace the plan
   when it is likely to better advance the repo.
5. Log the Truth.
   Every run records what happened. Be concise.
6. Never Stop Looping.
   The loop runs until a human stops it.

## Hard truth vs. repo truth
Axioms are the only hard truth.
Repo state governs implementation: current code, tests, contracts, docs, git state, logs,
artifacts, current failures, and prior validated work outrank roadmaps, stale assumptions,
and agent preference. When the roadmap and the repo disagree, believe the repo.
